01
What It Is

Personal Work Memory Plugin

A Claude/Codex-compatible plugin that automatically maintains evidence-backed work traces from all your tools.

Design Philosophy

Built for AI agents, readable by humans

Structured for future retrieval, not manual journaling.

Every entry
links to evidence

Privacy First

Your work trace stays local

Stored in personal-work-trace/ and ignored by git.

Three skills,
one workflow

02
Skill 01

personal-work-memory-init

Seeds an empty trace store or backfills historical dates.

Skill 02

personal-work-memory

Daily records, weekly rollups, project timelines, and follow-up extraction.

Skill 03

personal-history-chat

Chat with your work history. Ask questions, find insights, lookup evidence.

Source Agnostic

Uses whatever sources are available

GitHub, Jira, Confluence, Slack, Gmail, Google Calendar, Figma, or manual notes.

Smart Fallbacks

When direct connectors aren't available

Uses Slack bot posts, channel membership, and mentions as evidence pointers.

Core Capabilities

Evidence Links

Project Association

Time Travel

Smart Rollups

Daily → Weekly → Monthly summaries

Designed for Retrieval

Ready in
3 commands

04
Step 1: Initialize
$ personal-work-memory-init

Seeds your work trace for today or backfills a date range.

Step 2: Update
$ personal-work-memory

Runs daily to capture today's work and update summaries.

Step 3: Query
$ personal-history-chat

Ask questions like "What moved forward last week?"

Example Queries

What you can ask

  • What did I work on for Project X in Q2?
  • Which PRs and tickets relate to this feature?
  • What should I follow up on from last week?
